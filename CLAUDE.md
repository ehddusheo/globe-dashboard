# CLAUDE.md - AI Development Guide for RINDA Globe Dashboard

## Project Overview

AI-powered global market analysis SPA. Vanilla JS/CSS/HTML -- no frameworks, no build tools, no npm. Serves static files directly via GitHub Pages.

**Live**: https://ehddusheo.github.io/globe-dashboard/

---

## Architecture

### Core Files

| File | Lines | Purpose |
|------|-------|---------|
| `index.html` | ~140 | Single HTML entry point. Loads Globe.gl and TopoJSON from CDN via `<script>`. Contains all DOM structure. |
| `app.js` | ~2,700 | All application logic: globe init, UI interactions, AI wizard, PDF generation, i18n, ticker, rate limiting. |
| `styles.css` | ~1,200 | All styles. CSS custom properties on `:root`. Glass-morphism dark theme. Responsive breakpoints. |
| `data.js` | ~31,000 | **Auto-generated -- DO NOT edit manually.** Contains `INDUSTRIES`, `COUNTRIES`, and `TICKER_DATA` objects. Regenerated by the Python pipeline. |

### Key Globals in `app.js`

- `GEMINI_KEY` -- Gemini API key, base64-encoded in 4 segments and decoded at runtime via `atob()`
- `SHEET_WEBHOOK_URL` -- Google Apps Script webhook URL for logging analysis requests to Google Sheets
- `globe` -- Globe.gl instance
- `selectedCountry`, `selectedIndustry` -- current selection state
- `worldGeoJson` -- loaded TopoJSON geometry

### Key Globals in `data.js`

- `INDUSTRIES` -- object with 12 industry definitions (keys, Korean/English names, icons)
- `COUNTRIES` -- object with 149 country records (GDP, population, trade data, region, etc.)
- `TICKER_DATA` -- array of market ticker entries (indices, forex, commodities)

## Design System

Glass-morphism dark theme with command center / HUD aesthetic.

### CSS Custom Properties (`:root`)

```css
--primary: #00f0ff;        /* Cyan -- main accent */
--primary-dim: rgba(0,240,255,0.15);
--secondary: #7b2fff;      /* Purple */
--accent: #ff3366;         /* Pink -- CTA/alert */
--bg-dark: #060612;        /* Deep space background */
--bg-panel: rgba(8,12,35,0.88);  /* Glass panel fill */
--text: #d0d8ff;           /* Default text */
--text-bright: #ffffff;
--success: #00ff88;
--warning: #ffaa00;
--font-title: 'Orbitron', sans-serif;
--font-body: 'Rajdhani', 'Noto Sans KR', sans-serif;
```

### Styling Conventions

- Glass panels: `background: var(--bg-panel); backdrop-filter: blur(12px); border: 1px solid var(--primary-dim);`
- Use existing CSS custom properties -- do not introduce new colors without updating `:root`
- All animations use CSS `@keyframes` -- no JS animation libraries
- Font stack: Orbitron for titles/headings, Rajdhani for body, Noto Sans KR for Korean text

## Internationalization (i18n)

The app supports Korean (KO) and English (EN) via a language toggle button in the header.

- Language state is tracked in `app.js` (look for the lang toggle button handler and associated rendering logic)
- Industry and country names in `data.js` have both `name` (English) and `name_ko` (Korean) fields
- When adding new UI text, always provide both Korean and English versions
- The `#lang-toggle` button in the header switches between languages

## Mobile

- **Detection**: User agent check + `window.innerWidth < 768`
- **Bottom navigation**: `#mobile-nav` element in `index.html` with tab buttons: Globe, Search, Analysis, Data
- **Performance**: On mobile, globe disables bump/background images and atmosphere, uses lower renderer resolution
- **PDF handling**: Special logic for mobile WebView environments (Instagram, Threads, KakaoTalk, LINE, NAVER) -- generates blob and opens in new tab

## API Keys & Secrets

- **Gemini API key**: Base64-encoded in 4 parts at top of `app.js`. Decoded via `atob()` and concatenated at runtime. This is intentional for a client-side app with no backend.
- **Google Sheets webhook**: `SHEET_WEBHOOK_URL` constant in `app.js`. Points to a Google Apps Script deployment.
- No `.env` file. No server-side secrets management.

## Data Pipeline

Python scripts in `/scripts/` generate `data.js`. Run manually or via GitHub Actions daily cron.

### Pipeline Order (run via `scripts/run_all.py`)

1. `fetch_worldbank.py` -- World Bank API -> `raw_data/wb_countries.json` + `raw_data/wb_indicators.json`
2. `fetch_market.py` -- Yahoo Finance + ExchangeRate API -> `raw_data/market_data.json` + `raw_data/exchange_rates.json`
3. `generate_data.py` -- Merges all `raw_data/*.json` -> `data.js`

### Running the Pipeline

```bash
pip install -r requirements.txt   # requests, beautifulsoup4
python scripts/run_all.py
```

### GitHub Actions

- Workflow: `.github/workflows/daily-update.yml`
- Schedule: Daily at UTC 21:00 (KST 06:00)
- Commits as "Globe Data Bot" if data changed
- Can be triggered manually via `workflow_dispatch`

## Security Utilities

`app.js` includes built-in security helpers:

- `escHTML(str)` -- HTML entity escaping
- `safeUrl(url)` -- URL sanitization (only allows http/https)
- `isValidUrl(url)` -- URL validation
- `sanitizeFilename(name)` -- Filename sanitization for PDF downloads
- `createRateLimiter(ms)` -- Rate limiter factory. Used for:
  - Country lookup: 30s cooldown
  - AI analysis: 60s cooldown
  - Sheet webhook: 10s cooldown

## Common Tasks

### Add a new country
Edit the Python pipeline (`fetch_worldbank.py` country list), then run `scripts/run_all.py`. Do NOT edit `data.js` directly.

### Add a new industry
Update `generate_data.py` to include the new industry definition, then run the pipeline.

### Modify UI layout
Edit `index.html` for structure, `styles.css` for styling. The layout uses CSS Grid and Flexbox.

### Update AI prompt
Search `app.js` for the Gemini API call (look for `generativelanguage.googleapis.com`). The prompt template is constructed inline.

### Change color scheme
Update CSS custom properties in `:root` at the top of `styles.css`.

### Add new UI text
Always add both Korean and English versions. Follow the existing i18n pattern in `app.js`.

## Gotchas

- `data.js` is ~31,000 lines. Never edit it by hand -- always regenerate via the pipeline.
- The Gemini API key is client-side by design. Do not try to move it server-side -- there is no server.
- Globe.gl is loaded from CDN (`unpkg.com`). The CSP in `index.html` `<meta>` tag must allow it.
- The `Content-Security-Policy` meta tag in `index.html` restricts script/connect sources. If adding new external APIs or CDNs, update the CSP.
- CSS uses `overflow: hidden` on `body` -- the globe fills the viewport. Scrolling happens within panels only.
- All JS is in a single `app.js` file. There are no modules or imports (beyond `<script>` tags in HTML).
